# ブラウザ再試行ロジックの導入

「Edgeが見つかったが動作しない場合にChromeを試す」といった、複数のシステムブラウザを順次試行する仕組みを導入します。これにより、特定のブラウザが破損している環境での生存率を高めます。

## ユーザーレビューが必要な項目

- **試行順序**: 現在は Chrome を優先的に探し、次に Edge を探す設定になっています。この順序でリスト化し、先頭から順に起動を試みます。
- **エラー表示**: 最初のブラウザが失敗した際に警告を表示し、次のブラウザを試す旨をコンソールに通知します。

## 変更内容

### CrawlerService (ブラウザ操作サービス)

#### [MODIFY] [CrawlerService.cs](file:///C:/Users/uchij/Desktop/polite/Services/CrawlerService.cs)

- **`GetBrowserPath` を `GetAllBrowserPaths` へ変更**: 単一のパスを返すのではなく、見つかったすべての有効なブラウザパスを `List<string>` で取得するようにします。
- **`EnsureBrowserAsync` のループ化**: 
    - 取得したリストにあるブラウザを一つずつ `LaunchAsync` にかけます。
    - 起動に成功した時点でループを抜け、そのパスを以降のセッションで使用するように固定します。
- **フォールバックフローの整理**:
    1. システムブラウザ候補 (Chrome, Edge 等) を順に試す。
    2. 全滅した場合、`runtimes` フォルダの Chromium を試す。
    3. それでもダメな場合のみエラーとする。

## 確認計画

### 自動テスト / 動作確認
- コンソールログを確認し、「起動失敗 -> 次のブラウザへの切り替え」というフローが意図通り出力されるかを確認します。
- ブラウザパスにわざと無効な値を混ぜるなどして、再試行ロジックが機能することを確認します。

### 手動確認
- 用户の環境で、Edgeが優先的に見つかりつつも（何らかの理由で）動作しない場合に、Chrome等に自動で切り替わって動作が継続されることを確認いただきます。
