# 修復計画: CSV インポート処理の改善と targets.json 同期の堅牢化

CSV インポート（`--import`）を実行した際に `targets.json` が期待通りに作成されない、あるいは空になってしまう問題を修正します。また、BOM 付きファイルなどの読み込み耐性を向上させます。

## 修正が必要な点
- **BOM とトリミングの処理**: CSV ファイルの先頭に BOM（Byte Order Mark）がある場合、最初の URL が `http` で始まると判定されずスキップされてしまう問題を修正します。
- **インポート時の DB 同期**: `targets.json` が空の状態でインポートを開始した場合、DB 内の既存データが JSON に反映されない（上書きされてしまうように見える）問題を回避するため、インポート前にも DB からの読み込みを試行します。
- **既存エントリの更新**: 現在は URL が重複しているとスキップされますが、インポートによって設定（セレクタ等）を更新したい場合もあるため、既存エントリに対しても情報をマージするように改善します。

## 提案する変更内容

### [Program.cs](file:///c:/Users/uchij/Desktop/polite/Program.cs)

#### [MODIFY] `HandleImportCommandAsync`
- `File.ReadAllText` で読み込み、`Trim('\uFEFF')` で BOM を除去した後に処理を開始します。
- 同様に行の先頭の空白も考慮します。
- 処理の冒頭で `targets.json` が空の場合は DB から設定を読み込み、既存の監視対象と CSV の内容がマージされるようにします。
- URL が既存でも、セレクタなどが追加されている場合は更新を行い、`targets.json` に書き出す対象（`count`）に含めます。

## 検証計画
- BOM 付き UTF-8 形式の CSV を作成し、正しくインポートされるか確認。
- `targets.json` を削除した状態で `--import` を実行し、DB の既存データと新規 CSV データが合体した `targets.json` が出来上がるか確認。
- 重複する URL だがセレクタが異なる CSV でインポートを行い、情報が反映されるか確認。
