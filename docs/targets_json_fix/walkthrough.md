# 修正内容の確認 (Walkthrough): CSV インポートと JSON 同期問題の解消

CSV インポート（`--import`）時に設定が消失したり、`targets.json` が空になったりする問題を全面的に修正しました。

## 実施した主な変更

### 1. CSV インポートの堅牢化 (HandleImportCommandAsync)
- **BOM対応**: UTF-8 BOM（ファイルの先頭に付与される特殊な記号）が付いている CSV でも、1行目のデータを正しく読み込めるように修正しました。
- **データ保護 (DB同期)**: インポート開始時に、現在の DB の内容を自動的に読み込んで JSON とマージするようにしました。これにより、JSON が空の状態からインポートしても既存の設定が消えることはありません。
- **既存エントリの更新**: 重複する URL が CSV に含まれている場合、以前はスキップされていましたが、現在は CSV 側の新しい設定（セレクタ等）で既存のエントリを更新するように改善しました。

### 2. targets.json の整合性維持
- インポートや新しい情報の追加があるたびに、DB と `targets.json` の両方を確実に最新の状態に保つよう処理フローを見直しました。
- 空のファイルや不正な JSON 形式によるクラッシュを防ぐガードを全ての関連コマンドに追加しました。

## 検証結果
- [x] BOM（UTF-8）付きの CSV ファイルを正しく読み込み、`targets.json` に反映されることを確認。
- [x] インポート時に既存のデータが保持され、新しいデータが追加・更新されることを確認。
- [x] 空の `targets.json` から開始しても、DB の内容が保護された状態でインポートが完了することを確認。

## 使い方に関する注意
インポート元の CSV でセレクタが空の場合は既存のセレクタが維持されますが、新しいセレクタが記載されている場合は上書きされます。インポート前に `.\RssGenerator.exe --sync` を実行してバックアップ（JSON同期）をとっておくとより安全です。
