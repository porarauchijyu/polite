# 修復計画: 構成ファイル（targets.json）の配置とパス解決の改善

`targets.json` が期待した場所（プロジェクトルート）に存在しなかったり、実行環境によって作成場所が分散したりする問題を解決します。

## 修正が必要な点
- **ファイル位置の不整合**: 現在、設定ファイルが `bin\Debug\net48\` 内に作成されており、ルートディレクトリから消失しています。
- **相対パスの問題**: `"targets.json"` や `"feed.sqlite"` といった相対パスを直接使用しているため、コマンドを実行したディレクトリによって参照されるファイルが異なってしまいます。

## 提案する変更内容

### 1. 物理的なファイル配置の整理
- `bin\Debug\net48\targets.json` の内容を、プロジェクトルートの `targets.json` に復元・統合します。

### 2. [Program.cs](file:///c:/Users/uchij/Desktop/polite/Program.cs) 及び [DatabaseService.cs](file:///c:/Users/uchij/Desktop/polite/Services/DatabaseService.cs) の修正
- ファイルパスの解決を「実行ファイル（EXE）が存在するディレクトリ」を基準にするように変更、もしくは環境変数などを考慮した絶対パス解決を導入します。
    - 今回はシンプルに、実行場所に関わらず EXE と同じ場所（またはプロジェクトルート）のファイルを見に行くように修正します。

## 検証計画
- `targets.json` をルートに配置し、どのディレクトリから EXE を叩いても正しく同期・更新が行われることを確認します。
- ユーザー様がエディタで開いているファイルと、プログラムが実際に更新しているファイルが一致することを保証します。

## ユーザー様への確認事項
- 現在、どの場所の `targets.json` をご覧になっていますか？（例：プロジェクトのトップ、または `bin` フォルダの中など）
- 設定ファイル（json）やデータベース（sqlite）は、常にプログラム本体（EXE）の近くにあることを期待されますか？それともプロジェクトのルートにあることを期待されますか？
