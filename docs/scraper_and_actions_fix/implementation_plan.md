# 実装計画：スクレイパー改善とGitHub Actions修正

## 1. 概要
スクレイパーの日時リセットバグを修正し、ナビゲーションリンクなどの不要なデータを排除します。また、GitHub Actionsでのプッシュ失敗問題を解決し、安定した運用を可能にします。

## 2. 変更内容

### 2.1. スクレイパー (`web/scripts/scraper.ts`)
- **データ永続化**: 
    - 取得済みの記事データを読み込み、URLをキーとした Map で管理します。
    - スクレイピング中に一致する URL が見つかった場合、既存の `date` を再利用します。
- **フィルタリング**:
    - `junkWords` 配列に「プライバシーポリシー」「お問い合わせ」「English」などを追加します。
    - タイトルの文字数や特定のパターンによる除外を強化します。

### 2.2. GitHub Actions (`.github/workflows/deploy.yml`)
- **Git同期**: 
    - プッシュ前に `git pull --rebase origin main` を行うことで、ブランチの不整合を防ぎます。
    - `[skip ci]` タグをコミットメッセージに含め、無限ループを確実に防止します。

## 3. 影響範囲
- スクレイピングの結果生成される `articles.json` および各 RSS フィードファイル。
- リポジトリへの自動コミット履歴。
