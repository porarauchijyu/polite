# RSSフィードのサイト別保存機能の実装完了

全体のまとめフィード（`feed.xml`）を維持しつつ、登録されている各サイトごとの個別フィードを生成する機能を実装しました。

## 実施内容

### 1. RSS 生成ロジックの拡張 (`RssService.cs`)
- **`GenerateAllFeeds` メソッドの追加**: 全体フィードと個別フィードを一度に更新するバッチ処理を実装しました。
- **個別出力フォルダの作成**: 実行ファイルと同じ場所に `feeds` フォルダを自動作成し、そこに保存するようにしました。
- **ファイル名サニタイズ**: サイト名に含まれる不適切な文字（`/`や`:`など）を自動で `_` に置換する機能を追加しました。
- **フィード情報のカスタマイズ**: 個別フィードのタイトルや説明にサイト名を自動反映するようにしました。

### 2. CLI/GUI 巡回処理の更新
- **CLI (`Program.cs`)**: 巡回完了後、全てのフィードを一括生成するように変更。
- **GUI (`MainWindow.xaml.cs`)**: 全サイト巡回、および単独サイトの巡回完了後に全フィードを最新状態に更新するようにしました。

## 生成されるファイルの構成

修正後の実行環境では、以下のようなファイル構成になります：

```text
[実行ディレクトリ]
├─ feed.xml (従来通りの全サイトまとめ)
└─ feeds/
    ├─ feed_Osaka_Metro.xml
    ├─ feed_JR西日本.xml
    └─ ... (各サイトごとのファイル)
```

## 検証結果

- **ビルド確認**: `dotnet build` にて、エラーなしで正常に完了することを確認済み。
- **パス処理の安全性**: サイト名が空の場合や URL の場合でも、適切なファイル名が割り当てられることをロジック上で確認。

これにより、RSS リーダーなどで特定のサイトだけを個別に購読したいという要望に対応できるようになりました。
