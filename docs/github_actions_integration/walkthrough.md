# GitHub Actions 統合の修正内容確認 (Walkthrough)

GitHub の無料サーバー（GitHub Actions）で本アプリを自動実行するための設定と調整が完了しました。

## 変更内容

### 1. CI環境への対応 (`CrawlerService.cs`)
- GitHub Actions 等の CI 環境を自動検知するロジックを追加しました。
- 検知時には設定にかかわらず **強制的にヘッドレスモード（画面なし）** で起動するようにし、サーバー上でアプリが停止するのを防ぎます。

### 2. ファイル出力パスの調整 (`RssService.cs`)
- RSS フィード（`feed.xml` および `feeds/` フォルダ）の保存先を、実行ファイルと同じ階層ではなく **カレントディレクトリ（プロジェクトのルート）** になるように調整しました。これにより、GitHub Actions でのファイル収集が容易になります。

### 3. 自動巡回ワークフローの作成 (`.github/workflows/rss-generator-run.yml`)
- 以下の処理を自動で行うワークフローを作成しました：
    - **スケジュール実行**: 1時間ごとに自動起動します。
    - **ビルド**: ソースコードから最新の実行ファイルをビルドします。
    - **巡回**: `./RssGenerator.exe --run` を実行し、全サイトをチェック。
    - **自動保存**: 更新された `feed.sqlite` (DB) と `feed.xml` (RSS) をリポジトリに自動で `git commit & push` します。

## GitHub での使用手順

1.  **プライベートリポジトリの作成**: GitHub で新しいリポジトリ（Private推奨）を作成します。
2.  **コードの追加**: 現在のファイルをすべてリポジトリに追加（git push）してください。
3.  **自動実行の開始**: `git push` すると、1時間ごとに自動的に Actions が動き出します。手動で動かしたい場合は、GitHub の `Actions` タブから実行できます。
4.  **RSSフィードの利用**:
    - リポジトリ内に生成された `feed.xml` を直接参照するか、自動プッシュされたファイルを活用してください。

## 検証結果

- `CrawlerService` で `GITHUB_ACTIONS` 環境変数を正しくチェックできることをコード上で確認。
- ワークフローファイルが SDK-style のプロジェクト形式に対応していることを確認。
- RSS の出力パスがカレントディレクトリ相対になり、Actions で捕捉可能であることを確認。
